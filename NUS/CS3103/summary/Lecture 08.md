# Lecture 08: OSPF (Open Shortest Path First)

## Distance Vector (DV) Routing vs Link State (LS) Routing

### Distance Vector Routing

Router knows only cost to each destination which hides information and causing problems

### Link State Routing

Router knows entire network topology

## OSPF

Intradomain routing protocol based on link state routing. Its domain is also an Autonomous System (AS).

### Key Elements

- Topology dissemination
- Uses link state routing to compute shortest path

### Area Concept

Area Border Routers (ABRs) **summarize and distribute information of their area to other areas.**

**Autonomous System Border Routers (ASBRs)** take routes learned from other routing domains and redistribute them into OSPF domain as external routes.

Every router in an area computes the shortest path to every other router.

### Local Environment Informatino

- Neighboring routers
- Links to connected networks
- Cost of the links (metrics)

### Link-State Protocol

- Link-state packets are sent to all routers in the area
- Contain a list of network links (links to networks and routers) and their associated costs

### Neighbor Discovery and Maintenance

- Send neighbors with *Hello* messages
- The generated *Hello* messages include:
    - Router’s IP address and subnet mask
    - *Hello* interval
    - A list of neighbors whose *Hello*s the sender has already heard
- *Hello*s are multicasted to all OSPF routers using **244.0.0.5** every 10 seconds
- Failure to receive any *Hello* from a neighbor for 40 seconds assume that the link to that neighbor failed or that neighbor crashed

### Exchange of Link-State Information

- Each entity in an area distributes information about its local environment in packets called **Link State Advertisements (LSAs)**
- LSA’s are normally sent only under:
    - A router discovers a new neighbor
    - A link to a neighbor goes down
    - Cost of link changes
    - Basic refresh packets are sent every 30 minutes
- LSA’s are distributed to all routers in the area by **reliable flooding - to synchronize**
- Evert router in an area receive LSAs generated by other outers in that area
- The database of LSAs maintained by all routers in an area are set to be identical

### OSPF Packet Format

- ***Hello* packet**: `IP Header (20 B) | OSPF Header (24 B) | *Hello* packet body`
- **LSA packet**: `IP Header (20 B) | OSPF Header (24 B) | LSA Header (20 B) | LSA body`

### Type of OSPF packets

- *Hello*
- Database description
- Link state request
- Link State Advertisements (LSAs)
- Link state acknowledgement

### Avoiding traffic in transient link

- Transient link is a network with multiple routers
- A lots of LSAs messages are flooded to synchronize the LSA database if we use the template protocol
- To solve this, two routers will be elected as DR (Designated Router) and BDR (Backup Designated Router). Each router will have only BR and BDR as neighbor. The DR and BDR will have all routers as neighbors

### DB Initialization for a Newly Joining Router in Transient Link

1. DR sends a summary of its database LSAs to the new router - **database description packet**
2. The new router responds with a list of LSAs that it does not have or outdated - **link-state request packet**
3. DR forwards the full LSAs in the list to the new router - **link-state update packet**
4. New router respond with **link-state-ACK packet**
5. The newly joined router send its LSA to DR and BDR. DR and BDR multicast it to other routers.
